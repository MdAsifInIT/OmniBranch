# Phase 7 Context

Release readiness means all locally executable gates pass and release inputs are reproducible. Cross-OS CI, signed tags, provenance attestation, provider installations, and GitHub sandbox writes are external gates and remain unverified until evidence exists.

Dogfood is permitted only against disposable non-stable branches with the conservative policy pack. Release automation prepares artifacts but does not publish packages, tags, releases, pushes, or pull requests.
